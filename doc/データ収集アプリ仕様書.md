# データ収集アプリ仕様書 (collect_data.py)

## 1. 概要
本プログラム（`collect_data.py`）は、外部サイト（Football Lab）からJリーグの各チームの統計データを取得・加工し、予測モデルの学習および推論に使用できる形式のCSVファイルとして保存するためのデータ収集ツールです。

## 2. ディレクトリ構成
本プログラムの実行により、以下のディレクトリが管理・作成されます。
- `site_data/`: 外部サイトから取得した生のCSVデータを保存する一時的なフォルダ。
- `data/`: 加工・整形済みのCSVデータを保存するフォルダ（予測エンジンが参照する場所）。

## 3. 主要機能

### 3.1 起動引数による実行制御
コマンドライン引数により、柔軟な実行制御が可能です。
- **第1引数 (shotkey)**: 管理用識別子（必須）。出力されるCSVの `shotkey` 列に反映されます。
- **--force / -f**: キャッシュ（`site_data/` フォルダ内のファイル）を無視して、最新のWebデータを強制的に再取得します。

### 3.2 データ取得（スクレイピング）
- `pandas.read_html` を使用して、Football Lab のランキングページからデータを取得します。
- J1（j1001）とJ2（j1002）の両方のリーグデータを取得し、一つのデータセットに結合します。
- ネットワーク負荷軽減のため、一度取得したデータは `site_data/` に保存され、通常実行時はこれを利用します。

### 3.3 高度なチームマッピング（TeamIDMapper）
- Supabase から最新のチームマッピング情報を取得します。
- `difflib` を用いた曖昧検索（Fuzzy Matching）に加え、**検索結果をメモリ上にキャッシュ**する機能を備えており、大量のデータ処理を高速に行います。

### 3.4 汎用データ前処理
すべてのデータソースに対して、単一の汎用処理エンジン（`process_data`）が適用されます。
- `DATA_SOURCE_CONFIGS` 定義に基づき、自動的にカラムの読み替えと型変換を行います。
- 文字列（%記号など）の除去処理。
- 欠損値の `0.0` 埋め。
- 管理用の `shotkey` の一括付与。

## 4. データソース一覧
以下の10種類の指標を一元管理しています。

| 指標名 | URL種別 | 
| :--- | :--- |
| AGI,KAGI | チームの攻撃・守備の質 (kagi) |
| Attack Points | チャンスビルディングポイント 攻撃 (offence) |
| Capture Points | チャンスビルディングポイント 奪取 (gain) |
| Chance Construction Rate | チャンス構築率・シュート成功率 (chance) |
| Deffence Points | チャンスビルディングポイント 守備 (defense) |
| Dribble Points | チャンスビルディングポイント ドリブル (dribble) |
| Expected Goals | チームデータ ゴール期待値 (expected) |
| Goal Points | チャンスビルディングポイント ゴール (goal) |
| Pass Points | チャンスビルディングポイント パス (pass) |
| Save Points | チャンスビルディングポイント セーブ (save) |

## 5. 使用技術
- **Python 3.x**
- **Pandas**: データの取得、結合、加工。
- **Argparse**: コマンドライン引数の解析。
- **Logging**: 処理実行状況の可視化。
- **Difflib**: チーム名の曖昧マッチング。
- **Supabase (via db_client)**: マスタデータの参照。
